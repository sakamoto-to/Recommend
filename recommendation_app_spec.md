# 商品推薦アルゴリズム 学習アプリ 仕様書

## 概要

レコメンドシステムの数理的な仕組みをインタラクティブに学べるWebアプリ。
ユーザーがパラメータをスライダーやUIで操作しながら、コサイン類似度・協調フィルタリング・行列因子分解の計算過程と結果をリアルタイムで可視化する。

---

## 技術スタック

- **フレームワーク**: React（.jsx 単一ファイル）
- **グラフ描画**: Recharts
- **スタイリング**: Tailwind CSS（コアユーティリティのみ）
- **数値計算**: 純粋なJavaScript（NumPyは使わず）
- **アイコン**: lucide-react

---

## 画面構成

アプリは以下の3つのタブで構成される。タブ切り替えで各トピックに移動できる。

```
[ コサイン類似度 ] [ 協調フィルタリング ] [ 行列因子分解 ]
```

---

## タブ1：コサイン類似度

### 目的
2つのベクトルがどれだけ「似ているか」を角度で表す概念を直感的に理解する。

### UI要素

#### 入力パネル
- **ベクトルA**：スライダー2本（x成分: -5〜5、y成分: -5〜5）
- **ベクトルB**：スライダー2本（x成分: -5〜5、y成分: -5〜5）
- 各スライダーの現在値を数値で表示

#### 可視化パネル（2Dグラフ）
- 原点からベクトルAとベクトルBを矢印で描画
- 2つのベクトルのなす角θを扇形で表示
- 軸ラベル付きの座標平面（-6〜6）

#### 結果表示
- コサイン類似度の値（-1.0〜1.0）を大きく表示
- 数式の展開を段階的に表示：
  ```
  cos θ = (A・B) / (|A| × |B|)
        = (ax×bx + ay×by) / (√(ax²+ay²) × √(bx²+by²))
        = [計算結果]
  ```
- 類似度に応じたラベル表示：
  - 0.8〜1.0：「非常に似ている」
  - 0.5〜0.8：「やや似ている」
  - -0.5〜0.5：「あまり似ていない」
  - -1.0〜-0.5：「逆方向（正反対）」

#### ユーザーストーリー例
- 商品Aと商品Bの特徴ベクトル（価格帯・人気度など）をスライダーで設定して類似度を確認する

## タブ2：協調フィルタリング

### 目的
ユーザーの評価行列から「似たユーザー」を見つけ、未評価商品を予測する仕組みを体験する。

### UI要素

#### 評価行列の入力
- 5人のユーザー × 5つの商品 のグリッド（計25マス）
- 各セルは 1〜5 の評価値、または「未評価（空欄）」
- セルをクリックして評価値を入力・変更できる
- 初期値としてサンプルデータをプリセットで用意（「サンプルデータを読み込む」ボタン）

**サンプルデータ（初期値）：**

|       | 商品1 | 商品2 | 商品3 | 商品4 | 商品5 |
|-------|-------|-------|-------|-------|-------|
| ユーザー1 | 5 | 3 | - | 1 | - |
| ユーザー2 | 4 | - | 4 | 1 | 2 |
| ユーザー3 | - | 3 | - | 5 | 3 |
| ユーザー4 | 2 | 1 | 2 | 4 | - |
| ユーザー5 | - | 4 | 3 | - | 5 |

#### 対象ユーザー選択
- 「誰のレコメンドを見るか」をドロップダウンで選択（ユーザー1〜5）

#### ユーザー間コサイン類似度の表示
- 選択したユーザーと他の全ユーザーとの類似度を横棒グラフで表示
- 共通評価アイテムのみを使って計算
- 最も類似度が高いユーザーをハイライト表示

#### レコメンド結果
- 未評価商品に対して予測スコアを計算して表示
  - 計算方法：類似ユーザーの評価の加重平均（類似度を重みとして使用）
- 予測スコアの高い順に商品をリスト表示
- 「なぜこの商品がすすめられたか」の簡単な説明文を自動生成
  - 例：「ユーザー2があなたと最も似ており、商品3に4点をつけています」

## タブ3：行列因子分解

### 目的
評価行列をユーザー行列とアイテム行列に分解することで、隠れた特徴（潜在因子）を学習する仕組みを理解する。

### UI要素

#### パラメータ設定パネル
- **潜在因子数 k**：スライダー（1〜4、初期値2）
- **学習率 α**：スライダー（0.001〜0.1、対数スケール、初期値0.01）
- **正則化パラメータ λ**：スライダー（0.001〜0.1、初期値0.01）
- **学習ステップ数**：スライダー（10〜500ステップ、初期値100）

#### 「学習を実行する」ボタン
- クリックで勾配降下法を実行し、損失値の推移をリアルタイムアニメーションで描画

#### 損失関数の推移グラフ
- 横軸：ステップ数、縦軸：MSE損失値
- 折れ線グラフで学習曲線を表示

#### 行列の可視化
- **元の評価行列**（ヒートマップ）
- **再構成された評価行列（U × V^T）**（ヒートマップ）
- 2つを横並びで表示し、予測値との違いを直感的に確認できる
- ヒートマップの色：低評価→青、高評価→赤

#### 潜在因子の可視化（k=2の場合）
- ユーザー行列Uと商品行列Vを2次元散布図にプロット
- ユーザーは●、商品は★で区別
- 近い点同士が「相性がいい」ことを示す

#### 数式の説明
```
目標：R ≈ U × V^T
損失：L = Σ(r_ui - u_u・v_i)² + λ(||U||² + ||V||²)
更新：u_u ← u_u + α(e_ui × v_i - λ × u_u)
      v_i ← v_i + α(e_ui × u_u - λ × v_i)
```
- 各ステップで上記の更新式がどのように適用されるかをテキストで説明

## 共通UI仕様

### レイアウト
- 最大幅：1200px、中央寄せ
- ヘッダー：アプリ名「📊 レコメンド数理 学習アプリ」
- タブバー：3タブ、選択中タブはアクティブスタイル

### カラーテーマ
- 背景：ダークネイビー（#0f172a）
- カード背景：スレート（#1e293b）
- アクセント：シアン（#06b6d4）
- テキスト：白〜グレー系
- グラフ配色：青（ベクトルA）、オレンジ（ベクトルB）、緑（結果）

### レスポンシブ対応
- スマホ（〜768px）：縦1カラム
- PC（769px〜）：左右2カラム（入力 ／ 可視化）

### アニメーション
- タブ切り替え：フェードイン
- グラフ更新：スムーズな遷移（300ms）
- 行列因子分解の学習：損失グラフをリアルタイム更新

---

## 実装上の注意点

1. **数値計算はすべてJavaScriptで実装**（外部数値計算ライブラリ不使用）
2. **0除算の対策**：ベクトルのノルムが0の場合はコサイン類似度を0として扱う
3. **未評価データの扱い**：協調フィルタリングでは共通評価アイテムのみで計算する
4. **localStorageは使用しない**（状態はReactのuseStateで管理）
5. **単一ファイル（.jsx）で完結させる**
