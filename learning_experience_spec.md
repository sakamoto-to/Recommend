# 学習体験 改善仕様書（検討中）

## 現状の課題

現在の実装は「インタラクティブな数式ビューア」にとどまっている。
- スライダーを動かして結果が変わるのを眺めるだけ
- 数式の意味を能動的に考える機会がない
- 「わかった気になる」が「使える」にならない

---

## 目指す学習体験

> **「見る」→「予測する」→「確かめる」→「説明できる」** のサイクルを回す

---

## 学習モードの案

### モード A：ガイド学習モード（現状の強化）
今の「式の解説」アコーディオンを拡張する方向。

- 各ステップにアニメーションを付けて計算過程を「動かして見せる」
- 例：内積の計算を成分ごとに逐次ハイライト表示
- 「次へ」ボタンで段階的に式が展開される

**メリット**: 実装コストが低い、現UIを壊さない  
**デメリット**: まだ受動的。見るだけで終わる可能性がある

---

### モード B：予測チャレンジモード（推薦）
操作の前に「あなたはどう思う？」を挟む。

#### コサイン類似度タブ
- ベクトル A・B を表示し、スライダーを隠した状態でグラフだけ見せる
- 「この2つのベクトル、類似度はいくつだと思う？」→ スライダーで予測値を入力
- 送信後に正解と誤差を表示 → 「なぜその値になるか」を解説

#### 協調フィルタリングタブ
- 評価行列を見せて「ユーザー1は商品3を何点つけると思う？」
- 予測を入力 → アルゴリズムの予測値と比較
- 自分の直感とアルゴリズムのズレを体感させる

#### 行列因子分解タブ
- α・λ・k を変えたとき「損失はどう変化すると思う？」を予測させてから実行
- 「学習率を大きくしたら何が起きると思う？」

**メリット**: 能動的、誤りから学べる  
**デメリット**: 実装コストが高い、UXの設計が難しい

---

### モード C：穴埋め問題モード
式の一部を隠して入力させる。

```
cos θ = (A · B) / ( [?] × [?] )
```

- 選択肢方式（マルチプルチョイス）
- 数値入力方式（計算して埋める）
- 段階的ヒント機能

**メリット**: 式の定着に効果的  
**デメリット**: 問題・解答の設計コストが高い。難易度調整が難しい

---

### モード D：説明アウトプットモード（上級）
「この結果を自分の言葉で説明してみよう」

- 自由記述欄 → AI（Claude API）が採点・フィードバック
- 「なぜ類似度が高いのか説明してください」

**メリット**: 最も深い理解につながる  
**デメリット**: Claude API 連携が必要。スコープが大きい

- 現状の `ExplanationPanel` の文章・図解を全面的に見直す
- ステップアニメーション（計算過程を動かして見せる）を追加する

## 検討が必要な論点

1. **モードを切り替え方式にするか、タブ内に混在させるか**
   - 切り替え: 「学習モード / チャレンジモード」のトグル
   - 混在: インタラクティブ操作の途中に問いかけを差し込む

2. **進捗管理をするか**
   - 「コサイン類似度を理解した」チェックマークなど
   - localStorageで保存 or セッション内だけ

3. **難易度レベルを設けるか**
   - 初級：ガイド付き、公式の穴埋め
   - 中級：予測チャレンジ
   - 上級：パラメータチューニング、自由記述

4. **微積分・偏微分の章をどこに組み込むか**
   - 行列因子分解の「なぜこの更新式になるか」の直前に
   - 独立したタブとして設ける

## Phase 1 解説充実の方針（リサーチ結果より）

### コサイン類似度タブで追加すべき内容

| 追加項目 | 内容 |
|----------|------|
| 単位ベクトル化の図解 | ノルムで割ることで「長さが消え方向だけ残る」を可視化 |
| 投影の幾何学的意味 | 内積 = 「一方のベクトルを他方に投影した長さ × 長さ」を図示 |
| ユークリッド距離との対比 | 距離は「規模の差」、コサインは「方向の差」を具体例で示す |
| よくある誤解 | 「全部正の値だと類似度が高くなりやすい」「負の値の意味が不明確」 |

**参考**: [数学の景色](https://mathlandscape.com/cos-similar/) / [高校数学の美しい物語](https://manabitimes.jp/math/1378)

---

### 協調フィルタリングタブで追加すべき内容

| 追加項目 | 内容 |
|----------|------|
| 加重平均の分母の意味 | 「類似度の合計で割る＝重みを正規化する」を積み上げ棒グラフで示す |
| コサイン vs ピアソン相関 | 「全員が高評価した商品」ではコサインは高くなるがピアソンは差がつく |
| コールドスタート問題 | 新規ユーザー/アイテムへの限界を説明 |
| スパース性の説明 | 評価データが少ないとき何が起きるか |

**参考**: [Zenn CF解説](https://zenn.dev/datasciencekun/articles/33d3be4c4ce4ea) / [gihyo.jp](https://gihyo.jp/dev/serial/01/information-recommendation-system/0005)

---

### 行列因子分解タブで追加すべき内容

| 追加項目 | 内容 |
|----------|------|
| SVDとの違い | SVDは欠損値を0扱い → 「全部嫌い」と学習してしまう問題を図示 |
| 偏微分の導出 | ∂L/∂U[u] の計算過程を展開して更新式の「なぜ」を示す |
| λの効果の視覚化 | λ小（過学習）/ λ適切 / λ大（学習不足）の損失曲線3パターン比較 |
| メモリ効率 | 完全行列 O(nm) vs 因子分解 O((n+m)k) の比較 |

**参考**: [Zenn MF解説](https://zenn.dev/datasciencekun/articles/0ebe05c5dcdfff) / [Google ML Crash Course](https://developers.google.com/machine-learning/recommendation/collaborative/matrix)

### 難易度別カリキュラム構成（案）

```
初級（直感的理解）
  └─ コサイン類似度の「角度」としての理解
  └─ CFの「似た人の好みを参考にする」という発想
  └─ よくある誤解コーナー

中級（数学的導出）
  └─ 内積 → 正規化 → コサイン類似度の流れ
  └─ 加重平均の分子・分母の意味
  └─ 損失関数の2項の役割

上級（最適化理論）
  └─ SGD更新式の偏微分による導出
  └─ 正則化とハイパーパラメータのトレードオフ
  └─ 実務上の限界（コールドスタート、スパース性）
```
